# v0.7.0 vs v0.8.0 — Recursive Learning Framework Comparison

**Source:** GH issue #846 ("v0.8.0 ROADMAP — recursive learning + recursive
improvement framework: 10 ROI-ranked gaps"), pulled forward per operator
prime-directive memory `28860423-d12c-4959-bc8b-8fa9a94a33d9` (2026-05-18).

This document records the v0.7.0 substrate-state of each of the 10 ranked
gaps from #846 and indicates which were closed in the v0.7.0 ship window
versus which remain v0.7.1 / v0.8 follow-up work.

## Headline

**v0.7.0 already does the hard part:** depth-bounded signed synthesis +
skill promotion, end-to-end loop closure (3 phase results → 1 reflection
→ 6 Ed25519-signed `reflects_on` edges → 1 promoted skill). The loop is
real, working, and reproducible (see
`scripts/reproduce-recursive-learning.sh`).

## Gap-by-gap status

| Gap | Title | v0.8 plan | v0.7.0 status | Disposition |
|----:|---|---|---|---|
| 1 | Outcome-feedback weighting | `memory_mark_outcome` + curator weighting | Substrate primitives present (`reflects_on`, signed edges); no `mark_outcome` tool yet | **v0.7.1-blocker** — substantive new MCP tool surface |
| 2 | Skill discovery requires agent awareness | Curator scans reflections → `memory_pending` candidates | `memory_pending_*` exists; curator extension not wired to scan reflections | **v0.7.1-blocker** — curator behavioural extension |

| 4 | No `memory_skill_apply` | `memory_skill_apply(skill_id, current_context)` | `memory_skill_get` / `memory_skill_resource` ship; no apply convenience | **v0.7.1-blocker** — new tool |
| 5 | Reflection requires explicit source_ids | `memory_reflect_by_query(context=, namespace=)` | `memory_reflect` requires source_ids today | **v0.7.1-blocker** — convenience tool |

| 10 | MTP not live on current models | Issue #651 Phase 1 (mistralrs or candle backend) | `InferenceBackend` trait scaffolded in v0.7.0 (this initiative); GPU/MTP backend is a stub | **v0.7.1-blocker** — heavy work, gated by ML toolchain |

## What landed in v0.7.0 from this roadmap

- **#651** — `InferenceBackend` trait scaffold (`src/inference/mod.rs`) with
  Cpu (existing embeddings.rs + llm.rs) + Gpu (stub) variants. GPU returns
  `not implemented`; trait surface is callable today and unblocks the v0.8
  MTP / hot-path work referenced in Gap 10.
- **#654 (partial)** — supply-chain attestation MVP: SHA256 + (optional)
  signature verification of model files at load time. Plan documented in
  `docs/v0.7.0/inference-attestation.md`.
- **#791** — federation per-message Ed25519 signing header
  (`X-Memory-Sig`) with `AI_MEMORY_FED_REQUIRE_SIG=1` default in v0.7.0.
- **#224 / #311** — minimal `memory_share` MCP tool (point-to-point
  cross-agent copy into `_shared/<from>→<to>/`).
- **#518** — `session_id` parameter on `memory_recall` with per-session
  recently-accessed boost in the rerank scoring.
- **#718** — A2A harness cross-repo integration note
  (`docs/a2a-harness-integration.md`).

## What remains v0.7.1 / v0.8 work

- **#228** — E2E memory encryption (X25519 + ChaCha20). Requires schema
  bump + per-agent X25519 keypair plumbing; v0.7.1-blocker.
- **#519** — Promote `potential_contradictions` from advisory → blocking.
  Touches `src/mcp/tools/store.rs` (a per-session AVOID zone for this
  agent); v0.7.1-blocker.
- **#697** — Append-only Ed25519-signed forensic audit log. Overlaps
  with the existing `src/forensic/bundle.rs` export tarball; v0.7.1
  scope is to add the continuous `.jsonl.signed` write path and the
  `ai-memory audit verify --since DATE` subcommand.
