MCP Integration Primary Path

claude-memory runs as a Model Context Protocol (MCP) tool server over stdio. Configure it in Claude Code and memory operations become native tools -- no HTTP daemon needed.

Configuration

{
  "mcpServers": {
    "memory": {
      "command": "claude-memory",
      "args": ["--db", "/path/to/claude-memory.db", "mcp"]
    }
  }
}

Three-Tier Memory

Memories are organized into three tiers that mirror human memory systems. Each tier has automatic TTL management, and memories flow upward through access patterns.

Short

6h

Ephemeral context. Current task state, debugging notes. Extends +1h on each recall.

Mid

7d

Working knowledge. Sprint goals, recent decisions. Extends +1d on recall. Auto-promotes to long at 5 accesses.

Long

∞

Permanent. Architecture, user preferences, hard-won lessons. Never expires.

Recency Decay

Recent memories rank higher. The decay factor 1/(1 + days * 0.1) smoothly reduces relevance over time.

Ranking Formula

Recall uses a 6-factor composite score computed entirely in SQLite. Higher scores rank first.

score = fts_rank * -1 + priority * 0.5 + access_count * 0.1 + confidence * 2.0 + tier_boost + 1/(1 + days * 0.1)

Memory Lifecycle

Memories evolve through a lifecycle of creation, recall, promotion, linking, and consolidation.

Validation Pipeline

Every write path (CLI, HTTP API, MCP) passes through the same validation layer before touching the database.

Architecture

Single Rust binary, three interfaces, one shared database layer.

Quick Start

Option 1: MCP Server (Recommended)

# Install
cargo install --path .

# Add to Claude Code settings.json:
{
  "mcpServers": {
    "memory": {
      "command": "claude-memory",
      "args": ["--db", "claude-memory.db", "mcp"]
    }
  }
}

# Restart Claude Code -- 8 memory tools are now available

Option 2: CLI

# Store a memory
claude-memory store -T "Project uses Rust 2021" \
  -c "Rust edition 2021, Axum for HTTP, SQLite for storage." \
  --tier long --priority 7

# Recall it
claude-memory recall "what language and framework"

# Search with exact keywords
claude-memory search "Axum"

# List all memories
claude-memory list --json

Option 3: HTTP API

# Start daemon
claude-memory serve &

# Store via API
curl -X POST http://127.0.0.1:9077/api/v1/memories \
  -H 'Content-Type: application/json' \
  -d '{"title":"Test","content":"It works.","tier":"short"}'

# Recall
curl "http://127.0.0.1:9077/api/v1/recall?context=test"
