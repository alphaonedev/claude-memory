# v0.6.4 — AI NHI per-task starter prompts (top-shelf, self-contained)

**Companion to:** [`v0.6.4-roadmap.md`](v0.6.4-roadmap.md), [`v0.6.4-nhi-prompts.md`](v0.6.4-nhi-prompts.md), [`rfc-default-tool-surface-collapse.md`](rfc-default-tool-surface-collapse.md)
**Sprint window:** Mon 2026-05-04 → Fri 2026-05-08
**Release target:** Fri 2026-05-08 EOD

This deck has **one self-contained prompt per issue** (16 core + 3 stretch). The per-day grouping in [`v0.6.4-nhi-prompts.md`](v0.6.4-nhi-prompts.md) is the orchestration view; this file is the execution view. Pick a prompt by issue ID; load it cold into a fresh Claude Code (or compatible MCP-aware) session; the agent has every fact, file path, decision rule, acceptance criterion, gate, and handoff template it needs to ship that single issue.

## Conventions across all prompts

- **Working directory:** `ai-memory-mcp` repo root. PRs target `develop`, never `main`.
- **Branch:** every issue commits to `feat/v0.6.4-NNN-<slug>` cut from `feat/v0.6.4`.
- **Memory namespace:** `campaign-v064` for sprint state. Every prompt asks the agent to write a START + END handoff memory.
- **Commit format:** `<type>(<scope>): <summary>` per `CONTRIBUTING.md`. Sign all commits.
- **AI attribution:** every commit ends with `Co-Authored-By: <model> <noreply@anthropic.com>`.
- **Gates before PR:** `cargo fmt --check`, `cargo clippy -- -D warnings -D clippy::all -D clippy::pedantic`, `AI_MEMORY_NO_CONFIG=1 cargo test`, `cargo audit`. Zero failures.
- **PR body template:** Summary / Root cause / Test plan (checkboxes) / AI involvement section.
- **Hard prohibitions:** no commits to `main` or `release/*`; no skipping hooks; no force-push to protected branches; no global git config changes; no secrets in commits.
