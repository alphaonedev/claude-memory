## Why this release exists

v0.7.0 ships the attested-cortex (Ed25519 link signing, hooks, transcripts, AGE backend, capabilities v3, permissions/A2A). Three forward-looking items were called out during the v0.7.0 build but are explicitly out of scope for v0.7.0:

1. **MTP-Gemma-4 integration (`--llm-mtp` autonomous tier flag).** Google's Multi-Token Prediction enhancement to Gemma 4 (announced 2026-05) typically delivers 2-3× autoregressive throughput. ai-memory's autonomous tier (curator / contradiction / auto-tag / pre_recall G10 query expansion / R5 transcript extraction / R3 auto-link) is the single largest LLM consumer in the codebase. MTP unlocks a "real-time curator" mode and gives every G-track hook that calls an LLM more headroom against its EventClass timeout.

2. **G-track production replacements.** v0.7.0 ships reference-impl hooks (Jaccard for R3 auto-link, bag-of-words for R5 extraction). The production replacements are LLM-backed and deferred to v0.7.1 because (a) they need MTP to hit the latency budget, and (b) the v0.7.0 reference impls were the ship-gate, not the end state.

3. **Ultra-autonomous tier.** Shelved in v0.6 due to latency budgets. With MTP it becomes feasible. Bench-gated against the existing 50ms recall p95 invariant.

## Tasks

### MTP-1 — `--llm-mtp` autonomous-tier flag

**Spec:**
- Add `--llm-mtp` flag to the autonomous tier (or `[llm.autonomous] mtp = true` in config.toml). Default: off.
- When on, the Ollama / Candle client requests an MTP-aware Gemma 4 revision (specific HF revision TBD when Google publishes the weights).
- Fall back gracefully to standard Gemma 4 when the MTP revision isn't available locally.
- Surface in `ai-memory doctor` so operators can verify MTP is actually engaged.

**Bench gate:**
- Target: **2× curator throughput at p95** vs non-MTP autonomous tier.
- Workload: the canonical 1k-memory consolidation pass (existing v0.6 bench fixture).
- Gate placed in `bench.yml` as a hard-fail when MTP is reported as engaged but speedup < 1.5× (allows for unrelated regression headroom).

**Touched modules:** `src/llm.rs` (Ollama / Candle client), `src/config.rs` (autonomous tier config), `src/cli/doctor.rs` (engagement surface), `benches/curator_bench.rs` (new or extended).

**Cascade safety:** opt-in flag, no schema change, no MCP tool count cascade, no HookEvent variant.

### MTP-2 — production R3 auto-link hook

**Spec:** replace `tools/auto-link-detector/`'s Jaccard scorer with an LLM-graded similarity that calls the autonomous-tier client. Threshold becomes a confidence band, not a Jaccard ratio.

**Depends on:** MTP-1 (to hit per-store latency budget).

### MTP-3 — production R5 transcript extractor

**Spec:** replace `tools/transcript-extractor/`'s bag-of-words scorer with an LLM-driven extraction that mints standalone memories with `derived_from` links. The post-G11 mint companion (called out in I5's PR #590 body) lands here.

**Depends on:** MTP-1.

### ULTRA-1 — ultra-autonomous tier feasibility study

**Spec:** with MTP engaged, re-measure the latency budgets on the previously-shelved ultra-autonomous design (full LLM in the recall loop, not just curator). If p95 holds, ship it as `[features] ultra_autonomous = true`. If not, document the gap and table it again.

**Depends on:** MTP-1, MTP-2, MTP-3.

## Out of scope

- Anything that changes the v0.7.0 wire format (Ed25519 attestation, hook envelope shape, capabilities v3).
- New MCP tools (the v0.7.0 surface is canonical for the .x line).
- Schema migrations (v0.7.0 ends at v28; v0.7.1 stays at v28 unless we discover a hot bug).
