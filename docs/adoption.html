How this dashboard updates. An operator-configured cron job invokes scripts/update-adoption-metrics.sh on a daily cadence. The script polls the public APIs of GitHub, crates.io, npm registry, and (where available) Homebrew Analytics, then writes docs/adoption-metrics.json. This page reads that JSON via fetch() at load. The page never initiates any outbound request from the user's browser to a third party — the JSON is served from the same origin as the HTML, and the script itself is invoked only by the operator's cron. See docs/telemetry.md for the substrate-level no-phone-home commitment.

Schema. The JSON keys match the data-metric attributes on every cell above (dot-separated path, e.g. github.stars). Adding a new metric is a script change plus a single data-metric attribute on the new cell.
