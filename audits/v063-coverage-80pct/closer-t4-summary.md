## What Closer T4 did

Extracted the ~250-line TLS / mTLS block from `src/main.rs` into a new
`src/tls.rs` module. Verbatim move — every public symbol, every error
message, every comment is byte-for-byte identical with the W3 source.
Behaviour at the call sites is unchanged; only the path of the symbol
shifts from `main.rs` private items to `ai_memory::tls::*`.

Symbols moved:
- `load_rustls_config`
- `load_mtls_rustls_config`
- `load_fingerprint_allowlist`
- `rustls_pki_pem_iter_certs`
- `rustls_pki_pem_parse_private_key`
- `hex_short`
- `FingerprintAllowlistVerifier` (struct + `ClientCertVerifier` impl)
- `build_rustls_client_config`
- `DangerousAnyServerVerifier` (struct + `ServerCertVerifier` impl)

Call sites updated (4 in production code, 2 in `main.rs::tests`):
- `main.rs:1146` — `load_mtls_rustls_config` → `tls::load_mtls_rustls_config`
- `main.rs:1154` — `load_rustls_config` → `tls::load_rustls_config`
- `main.rs:3163` — `build_rustls_client_config` → `tls::build_rustls_client_config`
- `main.rs:4055`, `main.rs:4070` — pre-existing inline tests rerouted to `tls::load_fingerprint_allowlist`

`src/lib.rs` gained a single `pub mod tls;` line.
